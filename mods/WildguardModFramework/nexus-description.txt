[size=5]Wildguard Mod Framework (WMF)[/size]

The mod framework for Raiders of Blackveil. WMF gives players in-game control over their mods and gives mod authors a common API for discovery, game modes, and settings menus — with no engine restarts required.

[line]
[heading]Mod Discovery[/heading]

A [b]Mods[/b] button is added to the main menu and the in-game pause menu. Opening it shows every installed BepInEx plugin. Mods that support enable/disable have a working toggle — turn them off permanently or bring them back without touching the filesystem. Mods without toggle support are listed as well, so you always have a clear picture of what's loaded.

The enabled/disabled state is saved to a config file and applied at startup. New mods are detected automatically; uninstalled mods are cleaned up from the list.

[b]Multiplayer awareness[/b] — when hosting, [b]Allow Mods[/b] and [b]Allow Cheats[/b] toggles appear on the host setup screen. These only govern mods that are enabled in your config. The session name is suffixed with [b](cheats)[/b] or [b](modded)[/b] so other players know exactly what they're joining before they click.

Toggle changes are available from the main menu. The in-game Mods button shows current state but disables editing so nothing breaks mid-session.

[line]
[heading]Game Modes and Variants[/heading]

A [b]Game Mode[/b] stepper appears on the host setup screen and the solo start screen. Mods that register as game modes show up as selectable entries. Only one game mode can be active per session — selecting one disables all others. The default is [b]Normal[/b] (no game mode active).

Game mode mods can expose a single entry or multiple named variants from the same plugin (e.g. "Rogue Run" and "Rogue Run — Hard"). When [b]IsClientRequired[/b] is set, other players joining the session must also have the mod installed and enabled.

[line]
[heading]Mod Menus[/heading]

The Mods screen is split into two panels: a left bar for navigation and a right panel for content. Mods that expose a settings menu get their own named entry in the left bar — players can browse settings for each mod without leaving the game.

The menu system is aware of context: mods can disable or hide controls that are unsafe to change mid-session when the menu is opened from the pause screen.

[line]
[heading]Networking[/heading]

WMF exposes a reliable data channel for mods to exchange messages between the host and clients during a session — without registering new Fusion stream types or touching the game's network layer directly.

Mods subscribe to a named string channel and receive a byte payload whenever a message arrives on that channel. Messages can flow from the host to a specific client, from a client to the host, or broadcast from the host to every player running WMF. Delivery is guaranteed (Fusion reliable data) and routed through a single multiplexed stream, so mods stay isolated from each other and from the game's own traffic.

Full API reference is in the [url=https://github.com/fantastic-jam/raiders-of-blackveil-mods/blob/main/mods/WildguardModFramework/README.md]README[/url].

[heading]For Mod Authors[/heading]

WMF discovers mods automatically — no registration step needed. To unlock the full feature set (toggles, game modes, settings menus), implement [b]IModRegistrant[/b] on your plugin class, or use the duck-typed convention if you prefer not to ship an extra DLL.

Full API reference, code examples, and integration guide are on [url=https://github.com/fantastic-jam/raiders-of-blackveil-mods/blob/main/mods/WildguardModFramework/README.md]GitHub[/url]. The README is also included in the mod ZIP.
