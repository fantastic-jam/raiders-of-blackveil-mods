[size=5]ModManager[/size]

Gives the host control over which mods are active for a session.

When hosting, two extra toggles appear in the host setup screen — [b]Allow Mods[/b] and [b]Allow Cheats[/b] — each showing the list of active mods in that category on hover. Setting either to [b]No[/b] disables those mods before the session starts. The session name is automatically suffixed with [b](cheats)[/b] or [b](modded)[/b] so other players can see at a glance what kind of run they're joining.

Mods opt in by implementing the [url=https://github.com/fantastic-jam/raiders-of-blackveil-mods/releases?q=ModRegistry]ModRegistry[/url] interface (or the equivalent duck-typed convention — no DLL reference required).

[heading]Installation[/heading]
1. Install BepInEx (skip if already installed)
[list]
[*]Download [b]BepInEx 5[/b] from the [url=https://github.com/BepInEx/BepInEx/releases]BepInEx releases page[/url] — pick the BepInEx_win_x64 build[/*]
[*]Extract the contents into your game folder (the one containing RoB.exe)[/*]
[*]Launch the game once and close it — BepInEx will initialize its folder structure[/*]
[/list]
2. Install the mod
[list]
[*]Download ModManager and ModRegistry from the releases page[/*]
[*]Extract both ZIPs into your game's BepInEx folder[/*]
[*]Launch the game[/*]
[/list]
[line]
[heading]For mod authors[/heading]
Implement [b]IModRegistrant[/b] on your plugin class (or expose [b]GetModType()[/b] and [b]Disable()[/b] by convention — no DLL reference required). See the [url=https://github.com/fantastic-jam/raiders-of-blackveil-mods/tree/main/libs/ModRegistry]ModRegistry README[/url] for details.
