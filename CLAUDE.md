# CLAUDE.md

## What this repo is

C# BepInEx 6 mods for the game **Raiders of Blackveil**. Each mod is a separate VS project in one solution. Target: .NET Framework 4.6.2, C# 10, Release only ‚Äî there is no Debug config, no tests, no lint.

## Commands

Always use the `.bat` wrappers ‚Äî never invoke `.ps1` scripts directly.

```bash
./build.bat                                          # build all mods
./deploy.bat [ModName]                               # build + copy to local game
./release-mod.bat -ModName [ModName] -Version 1.2.3  # full release pipeline
./release-mod.bat -ModName [ModName] -Bump patch     # auto-increment patch/minor/major
./release-mod.bat -ModName [ModName] -Version 1.2.3 -DryRun   # preview only
./release-mod.bat -ModName [ModName] -Version 1.2.3 -SkipPush -SkipRelease  # local only
```

## Game source

`game-src/` is ILSpy-decompiled game source ‚Äî read it to understand APIs, types, namespaces. Do not edit it. Key namespaces:

- `RR` ‚Äî top-level: `AppManager`, `BackendManager`
- `RR.Game` ‚Äî gameplay: `Health`, `DashAbility`, `ChampionAbility`, `StatsManager`
- `RR.Game.Character`, `RR.Game.Damage`, `RR.Game.Stats` ‚Äî sub-namespaces
- `RR.Config` ‚Äî `PlayerSettings` (language, settings)
- `RR.UI.Components` ‚Äî `LocStringExt` (game's localization helper)

When in doubt about a type's namespace, grep `game-src/` before guessing.

## Mod structure

```
[ModName]/
‚îú‚îÄ‚îÄ [ModName]Mod.cs          # BepInEx plugin ‚Äî holds Id, Name, Version constant, Awake()
‚îú‚îÄ‚îÄ Patch/[ModName]Patch.cs  # Harmony patches
‚îú‚îÄ‚îÄ Assets/Localization/     # Translation files: [ModName].[lang].json
‚îú‚îÄ‚îÄ Config/                  # Optional .cfg templates (not overwritten on deploy)
‚îî‚îÄ‚îÄ [ModName].csproj         # imports Common.props + UserPaths.props
```

`Common.props` ‚Äî all shared assembly references. Add mod-specific refs directly in the mod's `.csproj`.

`UserPaths.props` ‚Äî machine-local game path, not in git, generated by `setup.ps1`.

## Harmony patches

- Resolve reflection fields once in `Apply()`, warn and bail if not found
- `AccessTools.Field` / `AccessTools.Method` for private member access
- Use `prefix` to intercept/block, `postfix` to observe/modify after

## Localization

Translation files live at `Assets/Localization/[ModName].[lang].json` (e.g. `PerfectDodge.en.json`). Format: flat `{"key": "value"}` JSON.

The localization class scans the plugin's `Assets/Localization/` directory at startup and loads all matching files. English is the fallback. Locale code comes from `AppManager.Instance?.PlayerSettings?.Gen_Language`.

`DataContractJsonSerializer` requires `UseSimpleDictionaryFormat = true` to parse flat JSON objects ‚Äî without it, it expects `[{Key, Value}]` pairs and silently fails.

## Releases

`release-mod.ps1` flow: bump `Version` constant ‚Üí build ‚Üí package ZIP ‚Üí `git commit` + tag ‚Üí push ‚Üí GitHub release.

- Tag format: `[ModName]-v[Version]`
- ZIP structure: `plugins/fantastic-jam-[ModName]/`
- Changelog is built from `git log --grep=[ModName]` since the previous mod tag ‚Äî **commit messages must contain the mod name** for changelog filtering to work
- Use `-DryRun` to preview the full plan including the generated changelog before touching anything

## Adding a mod

1. Create `[ModName]/[ModName].csproj` importing `Common.props` and `UserPaths.props`
2. Add to `raiders-of-blackveil-mods.sln`
3. `[BepInPlugin]`-annotated class inheriting `BaseUnityPlugin`, with a `public const string Version`
4. Harmony patches in `Patch/`
5. Add `[ModName]` to the `-ValidateSet` in both `release-mod.ps1` and `package-mod.ps1`
6. Create `[ModName]/metadata.json` ‚Äî add `nexus_mod_id` and `nexus_file_group_id` to enable automatic Nexus publish on release (omit the file to skip Nexus publishing for that mod)
