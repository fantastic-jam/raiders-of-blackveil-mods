# Enemy Targeting and Awareness

Extracted from `game-src/` — do not edit this file by hand.

```
NetworkCharacterBase          (RR.Game.Character/NetworkCharacterBase.cs)
  └─ NetworkNPCBase           (RR.Game.Character/NetworkNPCBase.cs)
       └─ NetworkEnemyBase    (RR.Game.Enemies/NetworkEnemyBase.cs)

            ├─ NetworkEnemyBoss

```

`NetworkChampionMinion` also extends `NetworkNPCBase` directly and overrides `SelectPlayerTarget` to always return null (minions have a different targeting model).

## Enemy state machine

`NetworkNPCBase._enemyState` (`EnemyState` enum, networked) drives everything:

```
Dormant → PreSpawn → Spawn → SearchingForTarget
  → ExecutingAbility → PostExecutingAbility → Idle → SearchingForTarget → …
```

The main loop runs in `NetworkNPCBase.FixedUpdateNetwork()`, which calls `UpdateTargetValidity()` then `FixedUpdateNPC()`.

`NetworkEnemyBase.FixedUpdateNPC()` calls `UpdateAggro()` to flip `_aggro`.

## Aggro / awareness

`NetworkEnemyBase.UpdateAggro()` — called every server tick while not yet aggro'd:
- Iterates `PlayerManager.Instance.GetPlayers()`
- Calls `GetPlayerThreatLevel(champion)` for each
- If any threat level > 0, calls `SetEnemyAggro()` on self
- Also notifies nearby (≤ 20 units) non-aggro enemies via `SetEnemyAggro()`

`GetPlayerThreatLevel(NetworkCharacterBase)`:
- Returns 0 if `playerCharacter == null || !playerCharacter.Stats.IsAlive`
- Adds **+1** if the player is within `_aggroRange` (default 15 units) of the enemy
- Adds the cumulative `damageDealt` by that player against this enemy
- Result > 0 triggers aggro

There is **no line-of-sight check** in the base aggro system. Proximity within `_aggroRange` is sufficient. There is no stealth check in `UpdateAggro` — the stealth check only happens at the target-validity and target-selection layers (see below).

## Target validity check

`NetworkNPCBase.UpdateTargetValidity()` — runs every tick before ability selection:
- Clears `_targetCharacter` if the object is invalid, dead, **or invisible** (`_targetCharacter.Stats.IsInvisible`)

This is the engine's built-in stealth break: if a target goes invisible mid-chase the enemy immediately loses it.

`NetworkNPCBase.IsTargetValid(StatsManager stats)` — same check as a predicate:
```csharp
return stats != null && stats.IsAlive && !stats.Health.IsInvisible;
```

## Target selection

`NetworkEnemyBase.SelectPlayerTarget(EnemyAbilityBase ability)` — the main selection method, called from `NetworkNPCBase.SelectAbility()` when `AbilityTargetType == Player`:

1. If no players remain, clear target and return false.
2. If enemy is **taunted** (`base.Taunter != null`), force `_targetCharacter = base.Taunter.Character` and exit.
3. Otherwise, switch on `ability.PlayerTargetSelectionType`:
   - `FarthestAway` → `GetPlayerCharacterFarthestAway()`
   - `HighestThreat` (default) → iterate players, skip invisible (`!playableChampion.Stats.Health.IsInvisible`), prefer highest `GetPlayerThreatLevel()`. There is a **10% hysteresis**: current target is only replaced if the challenger's threat is > 110% of current target's threat.
4. Returns `ability.AcceptsTarget(_targetCharacter)`.

`NetworkEnemyBase.GetAllPlayerCharactersTargets()` — used by `GeneralAttackConditionsOk()` to decide whether any attack is possible. Calls `IsPlayerTargetValid()` on every player's champion.

`NetworkEnemyBase.IsPlayerTargetValid(NetworkChampionBase)`:
- Returns false if there is a taunter that is not this character
- Delegates to `IsTargetValid(playerCharacter.Stats)` → alive + not invisible

Public setter: `NetworkEnemyBase.SetPlayerTargetCharacter(NetworkChampionBase)` — sets `_targetCharacter` directly.

## Mob movement to target

`NetworkNPCBase.PrepareForAbility()` moves the enemy toward `_targetCharacter` via NavMesh until it satisfies range + optional line-of-sight requirements of the current ability. `HasValidTarget()` (alive only, no invisibility re-check) gates this.

`NetworkEnemyDummyRanged` has its own completely independent targeting in `UpdateTargetDist()`: closest visible (non-invisible) player by Euclidean distance. This is a special-case non-ability-based enemy.

## Shameleon stealth

Ability: `ShameleonEnterTheShadowAbility` (`RR.Game.Character/ShameleonEnterTheShadowAbility.cs`)

On activation:
```csharp
base.Stats.Health.AddInvisible();
StealthTimer = TickTimer.CreateFromSeconds(runner, _stealthDuration * ActualDurationMultiplier);
```
`_stealthDuration` defaults to 4 seconds (configurable by perks via `ActualDurationMultiplier`).

Stealth is broken by:
- Timer expiry: `Health.RemoveInvisible()` called in `FixedUpdateNetwork()`
- Taking a hit (`CharacterEvent.OnHitWithAttack` or `OnHitWithPower`) while `StealthTimer.IsRunning`

`ShameleonShadowDanceAbility` also calls `AddInvisible` / `RemoveInvisible` — a second stealth ability exists.

**The canonical stealth check is:**
```csharp
playableChampion.Stats.Health.IsInvisible
// or equivalently:
playableChampion.Stats.IsInvisible
```
`Health.IsInvisible` is defined as `NetworkedInvisibleCounter > 0` (a networked integer counter, allowing multiple sources to stack).

## Champion identity

`NetworkCharacterBase.ChampionType` — `virtual ChampionType?`, returns `null` on the base class.

`NetworkChampionShameleon` overrides it:
```csharp
public override ChampionType? ChampionType => RR.Utility.ChampionType.Shameleon;
```

`ChampionType` is a simple enum in `RR.Utility`:
```csharp
public enum ChampionType { Ironhorn, Shameleon, Beatrice, Blaze }
```

To check if a `NetworkChampionBase` is Shameleon:
```csharp
champion.ChampionType == RR.Utility.ChampionType.Shameleon
```

## Recommended Harmony patch point for "Protect X"

**Patch: `NetworkEnemyBase.SelectPlayerTarget` — Prefix.**

Reason: this is the single method where all non-taunted, non-trivial player target selection happens for every standard enemy. It receives the ability being queued and writes `_targetCharacter`. A `Prefix` returning `false` can replace the entire selection logic cleanly.

Sketch of the override logic:
1. Skip if no protected target is configured (mod not active in this room).
2. If `base.Taunter != null` — do not override; the game's taunt system takes precedence (fall through to original).
3. Force `_targetCharacter = protectedPlayer.PlayableChampion`.
4. Check: if that champion's `ChampionType == ChampionType.Shameleon` **and** `champion.Stats.Health.IsInvisible` — clear `_targetCharacter` (enemy loses focus, fall through to original or return false).
5. Otherwise return `ability.AcceptsTarget(_targetCharacter)` as `__result` and return `false` (skip original).

`_targetCharacter` is a `protected` field on `NetworkNPCBase` — access it via `AccessTools.Field(typeof(NetworkNPCBase), "_targetCharacter")` resolved once in `Apply()`.

Note: `UpdateTargetValidity()` runs every tick independently and will also clear the target if the Shameleon goes invisible mid-chase, so the stealth break is already handled automatically. The only extra case to handle in the patch is preventing the mod from re-assigning an invisible Shameleon as the forced target at the start of each ability selection cycle.
